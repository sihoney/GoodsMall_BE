# Member Feature Spec

## 목표
- 회원 계정 수명주기 관리
- 인증과 토큰 재발급 제공
- 판매자 등록 관리
- 회원 신고 및 관리자 제재 프로세스 운영

## 기능 목록

### 1. 회원 가입
- 이메일 중복 검사
- 비밀번호 암호화 저장
- 기본 역할은 `USER`
- 기본 상태는 `ACTIVE`
- 가입 완료 후 `MemberSignedUpEvent` 발행

### 2. 로그인 및 토큰 관리
- 이메일/비밀번호 기반 로그인
- 액세스 토큰과 리프레시 토큰 발급
- Redis 기반 리프레시 토큰 저장
- 로그인 금지 제재 회원 차단

### 3. 회원 정보 조회/수정
- 내 정보 조회
- 내 정보 수정
- 특정 회원 조회/수정 API 제공
- 프로필 이미지 key 검증 및 URL 변환

### 4. 프로필 이미지 업로드 준비
- S3 presigned URL 발급
- 클라이언트 직접 업로드 방식 지원

### 5. 판매자 등록
- 일반 회원을 판매자로 전환
- 정산 계좌 정보 저장
- 회원 역할을 `SELLER` 로 변경

### 6. 회원 신고
- 자기 자신 신고 방지
- 동일 대상에 대한 중복 대기 신고 방지
- 신고 내역 조회 지원

### 7. 관리자 검토와 제재
- 신고 승인/반려
- 승인 시 선택적으로 제재 생성
- 활성 제재 중복 생성 방지
- 제재 비활성화 지원

## 비기능 고려사항
- 인증 상태는 게이트웨이와 JWT 규약을 공유한다.
- 제재 확인은 로그인 경로와 운영자 경로 모두에 영향을 준다.
- 신고 승인과 제재 생성이 연결되어 있어 관리자 UX 설계와도 결합된다.

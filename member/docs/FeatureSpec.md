# Member Feature Spec

## 목표
- 회원 계정 생명주기를 관리한다.
- 인증과 토큰 발급 기능을 제공한다.
- 판매자 등록 및 전환 기능을 관리한다.
- 회원 신고와 관리자 제재 프로세스를 운영한다.

## 기능 목록

### 1. 회원 가입
- 이메일 중복을 검사한다.
- 비밀번호를 안전하게 암호화하여 저장한다.
- 기본 역할은 `USER` 이다.
- 기본 상태는 `PENDING_VERIFICATION` 이다.
- 가입 완료 시 `MemberSignedUpEvent` 를 발행한다.

### 2. 로그인 및 토큰 관리
- 이메일/비밀번호 기반 로그인을 제공한다.
- 액세스 토큰과 리프레시 토큰을 발급한다.
- Redis 기반으로 리프레시 토큰을 저장한다.
- 로그인 제한 제재가 적용된 회원은 차단한다.

### 3. 회원 정보 조회/수정
- 내 회원 정보를 조회한다.
- 내 회원 정보를 수정한다.
- 특정 회원 조회/수정 API 를 제공한다.
- 프로필 이미지 key 를 검증하고 URL 로 변환한다.

### 4. 프로필 이미지 업로드 준비
- S3 presigned URL 을 발급한다.
- 클라이언트가 직접 업로드하는 방식을 지원한다.

### 5. 판매자 등록
- 일반 회원을 판매자로 전환할 수 있다.
- 정산 계좌 정보를 저장한다.
- 회원 역할을 `SELLER` 로 변경한다.

### 6. 회원 신고
- 자기 자신에 대한 신고를 방지한다.
- 동일 대상에 대한 중복 대기 신고를 방지한다.
- 신고 내역 조회를 지원한다.

### 7. 관리자 검토 및 제재
- 신고를 승인 또는 반려할 수 있다.
- 승인 시 선택적으로 제재를 생성할 수 있다.
- 활성 제재 중복 생성을 방지한다.
- 제재 비활성화를 지원한다.

## 비기능 고려사항
- 인증 상태와 게이트웨이 JWT 규약은 일관되게 공유되어야 한다.
- 제재 확인은 로그인 경로와 운영 경로 모두에 영향을 준다.
- 신고 검토와 제재 생성은 연결되어 있으므로 관리자 UX 설계에서 함께 고려한다.

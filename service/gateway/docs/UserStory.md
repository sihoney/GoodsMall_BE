# Gateway User Story

## 클라이언트
- 사용자로서, 여러 마이크로서비스를 하나의 베이스 URL 로 호출하고 싶다. 그래서 프론트엔드 구성을 단순하게 유지할 수 있다.
- 사용자로서, 로그인 후 받은 토큰 하나로 보호된 API 를 호출하고 싶다. 그래서 서비스별 인증 방식을 따로 신경 쓰지 않아도 된다.

## 백엔드 서비스
- 도메인 서비스로서, 인증 검증을 게이트웨이에 위임하고 싶다. 그래서 내부 서비스는 비즈니스 로직에 집중할 수 있다.
- 도메인 서비스로서, 요청 헤더로 회원 ID 와 역할을 받고 싶다. 그래서 공통 인증 컨텍스트를 쉽게 구성할 수 있다.

## 운영자
- 운영자로서, 각 서비스의 OpenAPI 문서를 게이트웨이 한 곳에서 보고 싶다. 그래서 테스트와 검토가 편해진다.
- 운영자로서, 공개 경로와 보호 경로를 설정으로 제어하고 싶다. 그래서 운영 정책을 빠르게 조정할 수 있다.

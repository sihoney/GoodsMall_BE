# Gateway Feature Spec

## 목표
- 서비스별 진입점을 하나의 엔드포인트로 통합한다.
- 인증을 중앙화해 각 도메인 서비스의 중복 구현을 줄인다.
- Swagger 문서를 한 화면에서 집계한다.

## 기능 목록

### 1. 서비스 라우팅
- Eureka 기반 `lb://` 라우팅
- 도메인별 path predicate 분리

### 2. JWT 인증 필터
- 비공개 API 요청 보호
- Access Token 만 허용
- 회원 ID 와 역할 정보를 헤더로 전달

### 3. 공개 경로 예외 처리
- 회원 가입, 로그인, 토큰 재발급
- 프로필 이미지 presign
- Swagger 문서 조회

### 4. Swagger 집계
- 각 서비스의 `/v3/api-docs` 를 게이트웨이 아래로 재노출
- `swagger-ui.html` 에서 멀티 서비스 문서 선택 가능

### 5. CORS 처리
- 로컬 프론트엔드 `http://localhost:5173` 허용
- 주요 HTTP 메서드와 모든 헤더 허용

## 운영 고려사항
- 경로 중복과 route id 중복이 없는지 점검 필요
- 인증 필터는 `/api/**` 기준이라 비표준 경로 추가 시 정책 보완이 필요

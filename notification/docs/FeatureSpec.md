# Notification Feature Spec

## 목표
- 회원이 후속 액션을 놓치지 않도록 도메인 이벤트를 알림으로 변환한다.
- 조회 API 는 단순하고 빠르게 유지한다.

## 기능 목록

### 1. 내 알림 목록 조회
- 페이지네이션 지원
- 최신순 정렬
- 공통 응답 래퍼 사용

### 2. 미읽음 개수 조회
- 배지 카운트 용도
- `member_id`, `is_read` 인덱스 사용

### 3. 알림 읽음 처리
- 본인 소유 알림만 처리 가능
- 읽음 상태를 `true` 로 전환

### 4. 이벤트 기반 알림 생성
- 자동 구매 확정 알림
- 주문 결제 성공 알림
- 주문 결제 실패 알림
- 판매자 정산 지급 성공 알림
- 판매자 정산 지급 실패 알림

## 운영 고려사항
- 알림 내용은 서비스 코드에 하드코딩되어 있어 다국어 대응 시 별도 분리가 필요하다.
- 참조 대상은 `referenceId + referenceType` 조합으로만 보관한다.
- 이벤트 중복 수신 방지 로직은 없어 upstream 보장이 중요하다.
